Counting Sort¶

Algoritma sorting yang ketiga adalah algoritma sorting yang memiliki waktu komputasi lebih rendah dibandingkan kedua algoritma sebelumnya (merge sort dan quick sort), yaitu counting sort. Pada algoritma ini, masing-masing data yang akan diurutkan, dihitung jumlahnya terlebih dahulu. Sehingga ketika jumlah masing-masing data telah diketahui, maka data-data ini akan mudah ditempatkan ke dalam indeks masing-masing. Misalkan terdapat data [4, 3, 1, 2, 4, 2, 1, 1], maka akan dilakukan terlebih dahulu, perhitungan jumlah masing-masing data, yaitu : data-'1' berjumlah tiga data, sehingga akan menempati posisi indeks ke-0 sampai dengan indeks ke-2 data-'2' berjumlah dua buah data, sehingga akan menempati indeks ke-3 sampai dengan ke-4. data-'3' berjumlah satu buah data, sehingga hanya akan menempati indeks ke-5, sedangkan data-'4' berjumlah dua buah data, dan kedua data ini akan menempati indeks ke-6 dan ke-7.

Loop kedua digunakan untuk menghitung jumlah elemen pada masing-masing data. Misalkan pada contoh sebelumnya, data '1' berjumlah tiga buah elemen. Dari loop kedua ini, dilakukan perhitungan komulatif dari masing-masing data. Loop terakhir akan data akan ditempatkan sesuai dengan nilai komulatif tersebut.

def countingSort(A,k):
C=[]
B=[]
for i in range(k+1):
C.append(0)
for j in range(len(A)):
C[A[j]]=C[A[j]]+1
B.append(0)
for i in range(1,k+1):
C[i]=C[i]+C[i-1]
for j in range(len(A)-1,-1,-1):
B[C[A[j]]-1]=A[j]
C[A[j]]=C[A[j]]-1
return(B)

Menghitung dan Mengurutkan¶

Algoritma Counting Sort mengurutkan array dengan menghitung berapa kali setiap nilai muncul.

Jalankan simulasi untuk melihat bagaimana 17 nilai bilangan bulat dari 1 hingga 5 diurutkan menggunakan Counting Sort.

Counting Sort tidak membandingkan nilai seperti algoritma pengurutan sebelumnya yang telah kita lihat, dan hanya bekerja pada bilangan bulat non-negatif.

Selain itu, Counting Sort cepat ketika rentang nilai yang mungkin k lebih kecil dari jumlah nilai N .

Cara kerjanya:

  1. Buat array baru untuk menghitung berapa banyak nilai yang berbeda.
  2. Telusuri array yang perlu diurutkan.
  3. Untuk setiap nilai, hitung dengan menambah angka pada array penghitung di indeks yang sesuai.
  4. Setelah menghitung nilai-nilai tersebut, telusuri array penghitungan untuk membuat array yang sudah diurutkan.
  5. Untuk setiap hitungan dalam larik penghitung, buat jumlah elemen yang tepat, dengan nilai yang sesuai dengan indeks larik penghitung.

Kondisi untuk Pengurutan Hitungan¶

Berikut adalah alasan mengapa algoritma Counting Sort dikatakan hanya berfungsi untuk rentang nilai bilangan bulat non-negatif yang terbatas:

  • Nilai bilangan bulat: Counting Sort bergantung pada penghitungan kemunculan nilai yang berbeda, sehingga nilainya harus berupa bilangan bulat. Dengan bilangan bulat, setiap nilai sesuai dengan indeks (untuk nilai non-negatif), dan terdapat jumlah nilai berbeda yang terbatas, sehingga jumlah kemungkinan nilai berbeda k tidak terlalu besar jika dibandingkan dengan jumlah nilainya N .
  • Nilai non-negatif: Counting Sort biasanya diimplementasikan dengan membuat array untuk penghitungan. Ketika algoritma memproses nilai-nilai yang akan diurutkan, nilai x dihitung dengan meningkatkan nilai array penghitungan pada indeks x. Jika kita mencoba mengurutkan nilai negatif, kita akan mengalami masalah saat mengurutkan nilai -3, karena indeks -3 berada di luar array penghitungan.
  • Rentang nilai terbatas: Jika jumlah nilai berbeda yang mungkin untuk diurutkan k lebih besar dari jumlah nilai yang akan diurutkan N Jika array penghitung yang kita butuhkan untuk pengurutan akan lebih besar daripada array asli yang perlu diurutkan, maka algoritma tersebut menjadi tidak efektif.

Mengimplementasikan Counting Sort di Python¶

Untuk mengimplementasikan algoritma Counting Sort dalam program Python, kita membutuhkan:

  1. Sebuah array berisi nilai-nilai yang akan diurutkan.
  2. Metode 'countingSort' yang menerima array bilangan bulat.
  3. Sebuah array di dalam metode untuk menyimpan jumlah nilai.
  4. Sebuah perulangan di dalam metode yang menghitung dan menghapus nilai, dengan menambah elemen dalam larik penghitung.
  5. Sebuah perulangan di dalam metode yang membuat ulang array dengan menggunakan array penghitung, sehingga elemen-elemennya muncul dalam urutan yang benar.

Satu hal lagi: Kita perlu mencari tahu nilai tertinggi dalam array tersebut, agar array penghitung dapat dibuat dengan ukuran yang tepat. Misalnya, jika nilai tertinggi adalah 5, array penghitung harus memiliki total 6 elemen, agar dapat menghitung semua kemungkinan bilangan bulat non-negatif 0, 1, 2, 3, 4, dan 5.

Kode yang dihasilkan akan terlihat seperti ini:

def countingSort(arr):
  max_val = max(arr)
  count = [0] * (max_val + 1)

  while len(arr) > 0:
    num = arr.pop(0)
    count[num] += 1

  for i in range(len(count)):
    while count[i] > 0:
      arr.append(i)
      count[i] -= 1

  return arr

mylist = [4, 2, 2, 6, 3, 3, 1, 6, 5, 2, 3]
mysortedlist = countingSort(mylist)
print(mysortedlist)

Kompleksitas Waktu Pengurutan Hitungan¶

Kecepatan algoritma Counting Sort berjalan bergantung pada rentang nilai yang mungkin. k dan jumlah nilai N .

Secara umum, kompleksitas waktu untuk Counting Sort adalah HAI ( N + k ) .

Dalam skenario terbaik, rentang nilai berbeda yang mungkin terjadi adalah... k sangat kecil dibandingkan dengan jumlah nilai. N dan Counting Sort memiliki kompleksitas waktu HAI ( N ) .

Namun dalam skenario terburuk, rentang nilai berbeda yang mungkin terjadi k ukurannya sangat besar dibandingkan dengan jumlah nilai N dan Counting Sort dapat memiliki kompleksitas waktu HAI ( N 2 ) atau bahkan lebih buruk.

Grafik di bawah ini menunjukkan seberapa besar kompleksitas waktu untuk algoritma Counting Sort dapat bervariasi.

Seperti yang Anda lihat, penting untuk mempertimbangkan rentang nilai dibandingkan dengan jumlah nilai yang akan diurutkan sebelum memilih Counting Sort sebagai algoritma Anda. Selain itu, seperti yang disebutkan di bagian atas halaman, perlu diingat bahwa Counting Sort hanya berfungsi untuk nilai bilangan bulat non-negatif.

Seperti yang telah disebutkan sebelumnya: jika angka yang akan diurutkan memiliki nilai yang sangat bervariasi (besar) k ), dan hanya ada sedikit angka yang perlu diurutkan (kecil) N ), algoritma Counting Sort tidak efektif.

Jika kita berpegang teguh N Dan k Dengan tetap mempertahankan nilai tetap, alternatif "Acak", "Menurun", dan "Menaik" dalam simulasi di atas menghasilkan jumlah operasi yang sama. Ini karena hal yang sama terjadi dalam ketiga kasus tersebut: Sebuah array penghitung disiapkan, angka-angka dihitung, dan array baru yang sudah diurutkan dibuat.

Counting Sort GFG¶

Counting Sort adalah algoritma pengurutan yang tidak berbasis perbandingan . Algoritma ini sangat efisien ketika rentang nilai input kecil dibandingkan dengan jumlah elemen yang akan diurutkan.

  • Ide dasar di balik Counting Sort adalah menghitung frekuensi setiap elemen unik dalam array input dan menggunakan informasi tersebut untuk menempatkan elemen-elemen tersebut pada posisi yang benar sesuai urutannya.
  • Ini berfungsi dengan baik ketika rentang elemen input kecil dan sebanding dengan ukuran array. Misalnya, untuk input [1, 4, 0, 2, 1, 1], ukuran array adalah 6 dan rentang elemennya dari 0 hingga 4.
  • Jika rentang array input berukuran lebih besar dari n Log n, di mana n adalah ukuran array, maka kita dapat mengurutkan array dengan lebih baik menggunakan algoritma pengurutan berbasis perbandingan standar seperti Merge Sort.

Algoritma Pengurutan Hitungan

  • Deklarasikan array hitungan cntArr[] dengan ukuran max(arr[])+1 dan inisialisasi dengan 0 .
  • Telusuri array input arr[] dan petakan setiap elemen arr[] sebagai indeks array cntArr[] , yaitu, jalankan cntArr[arr[i]]++ untuk 0 <= i < N .
  • Hitung jumlah prefiks pada setiap indeks cntArr[] .
  • Buat array ans[] dengan ukuran N.
  • Telusuri array arr[] dari akhir dan perbarui ans[ cntArr[ arr[i] ] - 1] = arr[i] . Juga, perbarui cntArr [ arr[i] ] = cntArr[ arr[i] ]- - .

Mengapa kita menghitung Jumlah Awalan?

Kita bisa saja menghitung kemunculan semua elemen dan satu per satu memasukkannya ke dalam array keluaran, tetapi kita menghitung jumlah awalan untuk mencapai stabilitas dalam algoritma. Perhatikan bahwa, setelah membangun jumlah awalan cntArr[], kita menelusuri array dari ujung kanan untuk memastikan bahwa kemunculan terakhir berpindah ke posisi yang benar terakhir dalam array yang diurutkan.

Cara Kerja Penghitungan dan Pengurutan

def count_sort(arr):
    if not arr:
        return []

    n = len(arr)
    maxval = max(arr)

    # create and initialize cntArr
    cntArr = [0] * (maxval + 1)

    # count frequency of each element
    for v in arr:
        cntArr[v] += 1

    # compute prefix sums
    for i in range(1, maxval + 1):
        cntArr[i] += cntArr[i - 1]

    # build output array
    ans = [0] * n
    # iterate in reverse to keep it stable
    for i in range(n - 1, -1, -1):
        v = arr[i]
        ans[cntArr[v] - 1] = v
        cntArr[v] -= 1

    return ans

    ans = count_sort(arr)
    print(ans)

Analisis Kompleksitas Pengurutan Hitungan (Counting Sort):

  • Kompleksitas Waktu : O(N+M) dalam semua kasus, di mana N dan M masing-masing adalah ukuran inputArray[] dan countArray[] .
  • Ruang Tambahan: O(N+M), di mana N dan M adalah ruang yang ditempati oleh outputArray[] dan countArray[] masing-masing.

Keunggulan Penghitungan Sortir:

  • Counting sort umumnya bekerja lebih cepat daripada semua algoritma pengurutan berbasis perbandingan, seperti merge sort dan quicksort, jika rentang input sebanding dengan jumlah input.
  • Algoritma Stabil

Kelemahan Pengurutan Berdasarkan Jumlah:

  • Tidak berfungsi pada nilai desimal.
  • Tidak efisien jika rentang nilai yang akan diurutkan sangat besar.
  • Ini bukan algoritma pengurutan langsung (in-place) , algoritma ini menggunakan ruang tambahan untuk mengurutkan elemen-elemen array.

Penerapan Metode Pengurutan Berdasarkan Jumlah:

  • Ini adalah algoritma yang umum digunakan untuk kasus-kasus di mana kita memiliki item dengan rentang terbatas. Misalnya, mengurutkan siswa berdasarkan nilai, mengurutkan acara berdasarkan waktu, hari, bulan, tahun, dan lain sebagainya.
  • Ini digunakan sebagai subrutin dalam Radix Sort.
  • Konsep pengurutan berdasarkan jumlah digunakan dalam Bucket Sort untuk membagi elemen ke dalam beberapa wadah yang berbeda.

def counting_sort(arr):
    if not arr: return []
    max_val = max(arr)
    # Array untuk menampung frekuensi
    count = [0] * (max_val + 1)
    
    for num in arr:
        count[num] += 1
    
    # Membangun kembali array yang terurut
    sorted_arr = []
    for num in range(max_val + 1):
        sorted_arr.extend([num] * count[num])
    return sorted_arr

def stable_counting_sort(arr):
    n = len(arr)
    max_val = max(arr)
    count = [0] * (max_val + 1)
    output = [0] * n
    
    for x in arr: count[x] += 1
    for i in range(1, len(count)): count[i] += count[i-1]
    
    # Isi output dari kanan ke kiri untuk kestabilan
    for i in range(n - 1, -1, -1):
        output[count[arr[i]] - 1] = arr[i]
        count[arr[i]] -= 1
    return output

print(f"Stable Sort: {stable_counting_sort([1, 4, 1, 2, 7, 5, 2])}")

def counting_sort_negatif(arr):
    min_val, max_val = min(arr), max(arr)
    range_val = max_val - min_val + 1
    count = [0] * range_val
    
    for x in arr: count[x - min_val] += 1
    # ... (lanjutkan rekonstruksi)
    sorted_arr = []
    for i, freq in enumerate(count):
        sorted_arr.extend([i + min_val] * freq)
    return sorted_arr

def comprehensive_counting_sort(arr):
    if not arr: return []
    # 1. Menentukan rentang
    min_val, max_val = min(arr), max(arr)
    range_val = max_val - min_val + 1
    count = [0] * range_val
    output = [0] * len(arr)
    
    # 2. Menghitung frekuensi
    for x in arr: count[x - min_val] += 1
    
    # 3. Akumulasi posisi
    for i in range(1, len(count)): count[i] += count[i-1]
    
    # 4. Membangun output
    for i in range(len(arr) - 1, -1, -1):
        output[count[arr[i] - min_val] - 1] = arr[i]
        count[arr[i] - min_val] -= 1
    return output

print(f"Final Result: {comprehensive_counting_sort([10, 4, 10, 2, 2, 5])}")
