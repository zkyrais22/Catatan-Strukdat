STACK¶

Struktur Data Struktur data merupakan cara programmer untuk merepresentasikan suatu data, dan memudahkan programmer untuk menyelesaikan permasalahan dengan menggunakan struktur data tersebut.

Struktur data Linear Structure merupakan struktur data sederhana dimana ketika data ditambahkan maka data tersebut berada pada posisi relatif terhadap data yang lain dari struktur data tersebut. Linear structure ini memiliki dua buah ujung. Dua buah ujung ini memiliki beberapa istilah seperti left-right, front-rear, ataupun top-bottom. Terdapat beberapa struktur data yang termasuk pada linear structure ini, yaitu :

  • Stacks
  • Queues
  • Lists

Ketiga jenis struktur data tersebut berbeda dalam penambahan atau penghapusan suatu data. Misalkan, suatu struktur data hanya diperbolehkan menghapus data pada salah satu ujung, sedangkan pada struktur data lain, penghapusan data dapat dilakukan pada kedua ujung.

Struktur data yang akan dibahas pertama kali adalah Stacks, yang terdiri dari :

  1. Stacks
  2. Operasi Stacks
  3. Contoh Implementasi Stacks
  4. Ekspressi Aritmatik Infix, Prefix, Postfix

Definisi Stacks¶

Stacks adalah satu struktur data dimana penambahan dan penghapusan data, hanya dapat dilakukan pada satu ujung yang sama, atau yang biasa dikenal dengan istilah top.

Semakin data jauh berada dari posisi top, maka data tersebut diindikasikan berada di stack lebih lama dibandingkan dengan data yang berada dekat pada data di posisi top.

Jika terdapat data baru yang ditambahkan di stack, maka data ini pulalah yang akan dihapus ketika terdapat proses penghapusan data. Konsep ini dikenal dengan nama LIFO-Last In First Out.

Konsep stack ini dapat ditemui pada permasalahan sehari-hari, misalkan tumpukan buku pada

Tumpukan buku

Jika kita ingin mengambil buku pada tumpukan buku tersebut, maka buku yang dapat kita ambil adalah buku yang berada di posisi teratas. Jika ingin mengambil buku yang berada di posisi paling bawah, maka kita harus mengambil buku-buku yang berada di posisi atasnya terlebih dahulu. Begitu juga ketika terdapat penambahan buku baru, maka buku baru ini akan berada di posisi paling atas.

Operasi pada Stacks¶

Terdapat operasi dasar pada Stacks,

  • stack (), inisialisasi stack yang kosong
  • push(data), penambahan data baru pada posisi top dari stack
  • pop(), penghapusan data yang terdapat di posisi top dari stack. R eturn value dari fungsi ini adalah data yang dihapus dari stack tersebut
  • peek(), informasi data yang terletak pada posisi top
  • isEmpty(), untuk memeriksa apakah stack dalam keadaan kosong
  • size(), informasi jumlah data yang terdapat pada stack

def reverseWord(word):
    # 1. Inisialisasi stack kosong
    dataStack = []
    
    # 2. PUSH: Masukkan setiap karakter ke dalam stack
    for ch in word:
        dataStack.append(ch)
    
    temp = ''
    
    # 3. POP: Ambil karakter dari atas stack sampai habis
    # Stack bersifat LIFO (Last In First Out), jadi yang terakhir masuk akan keluar duluan
    while len(dataStack) > 0:
        temp = temp + dataStack.pop()
        
    return temp

Delimiter Matching¶

Parentheses sering digunakan untuk urutan penyelesaian dalam persamaan Matematika, seperti contoh berikut:

P = 5 x (4 + 5) / ((3 + 2) x (10 - 8))

Dengan penggunaan parentheses maka persamaan tersebut dapat diselesaikan secara benar, hanya saja terkadang parentheses yang ditulis tidak lengkap, sehingga hasil persamaan tidak dapat dikerjakan atau tidak sesuai dengan yang diinginkan, seperti contoh berikut :

P = 5 x (4 + 5) / ((3 + 2) x (10 - 8)

Oleh karena itu diperlukan algoritma untuk pemeriksaan parentheses ini atau secara umum delimiter, yaitu '{ }', '[ ]', dan '( )'.

Beberapa tahapan atau algoritma untuk pengecekan parentheses ini adalah :

  1. Baca setiap karakter yang terdapat pada suatu string matematika
  2. Jika karakter adalah kurung buka, maka masukkan dalam stack
  3. Jika karakter adalah kurung tutup, ada beberapa kondisi yang harus diperiksa, yaitu :
  • Jika stack dalam keadaan empty, maka jumlah kurung tutup lebih banyak daripada kurung buku
  • Jika stack dalam keadaan tidak empty, maka maka pop stack, dan cocokkan karakter hasil pop dengan kurung tutup, jika sejenis, maka matched, jika tidak maka jenis kurung tidak sama
  1. Jika semua karakter telah terbaca, dan stack masih dalam keadaan terisi (tidak empty), maka jumlah kurung buka lebih banyak daripada kurung tutup

def desimal_ke_biner(desimal):
    # Menangani kasus khusus jika input adalah 0
    if desimal == 0:
        return "0"
        
    stack = []
    
    # Langkah 1: Push sisa bagi ke dalam stack
    while desimal > 0:
        sisa_bagi = desimal % 2
        stack.append(sisa_bagi) # Push ke stack
        desimal = desimal // 2  # Pembagian bulat dengan 2
        
    hasil_biner = ""
    
    # Langkah 2: Pop elemen dari stack untuk membentuk string biner
    while len(stack) > 0:
        hasil_biner += str(stack.pop()) # Pop dari stack
        
    return hasil_biner

Ekspressi Aritmatik Infix, Prefix, Postfix¶

Eksperssi aritmatik yang sering ditemui adalah operasi aritmatik infix. Pada operasi aritmatik infix ini, operator (+,-,*,/, dll) berada diantara dua buah operand (bilangan). Ekspressi aritmatik ini dapat dengan mudah diselesaikan ketika tidak ada ambiguitas, seperti:

A + B

A ∗ X

A − B + C

Akan tetapi, ekspressi aritmatik ini akan sulit diselesaikan jika terjadi ambiguitas, seperti ekspressi aritmatik berikut ini:

A + B ∗ C

Pada ekspressi tersebut, terdapat ambiguitas, yaitu apakah A + B diselesaikan terlebih dahulu ataukah B * C yang harus diselesaikan terlebih dahulu. Ambiguitas ini dapat diatasi dengan konsep operator precedence. Pada operator precedence, operator-operator dibagi menjadi beberapa level berdasarkan prioritas urutan penyelesaian.

Pada contoh ekspressi aritmatik A + B * C, berdasarkan operator precedence, maka urutan penyelesaian adalah B * C kemudian hasilnya ditambahkan dengan A. Permasalahan terjadi ketika pada ekspressi aritmatik A + B * C, A + B ingin diselesaikan terlebih dahulu. Untuk penyelesaian ekspressi aritmatik yang tidak sesuai dengan urutan operator precedence, maka tanda '( )' sangatlah diperlukan, sehingga operasi tersebut menjadi (A + B) * C.

Tanda '( )' tidaklah diperlukan pada ekspressi aritmatik prefix dan postfix. Pada ekspressi aritmatik prefix, operator mendahului operand, sedangkankan ekspressi aritmatik postfix, operator ditulis setelah operand.

Konversi Infix ke Postfix¶

Berikut algoritma untuk konversi ekspressi aritmatik infix ke postfix :

  1. Buat struktur data stack untuk menampung operator

  2. Baca ekspressi aritmatik dari kiri ke kanan tiap token :

  • jika token yang dibaca adalah operand maka masukkan operand tersebut ke dalam output string

  • jika token yang dibaca adalah kurung buka maka push kurung buka tersebut ke dalam stack

  • jika token yang dibaca adalah kurang tutup maka pop stack semua token sampai ditemukan kurung buka

  • jika token yang dibaca adalah operator maka :

    • pop operator-operator yang memiliki precedence lebih tinggi atau sama dan masukkan operator tersebut ke dalam output string

    • push token operator ke dalam stack

  1. Jika masih terdapat operator pada stack, maka pop operator yang tersisa dan letakkan pada output string

def infix_ke_postfix(ekspresi):
    # Menentukan tingkat prioritas operator
    prioritas = {'+': 1, '-': 1, '*': 2, '/': 2, '^': 3}
    
    stack = []      # Stack untuk menyimpan operator
    postfix = []    # List untuk menyimpan hasil postfix
    
    # Menghilangkan spasi untuk mempermudah pembacaan karakter
    ekspresi = ekspresi.replace(" ", "")
    
    for karakter in ekspresi:
        # 1. Jika karakter adalah operand (huruf atau angka), langsung masukkan ke hasil
        if karakter.isalnum():
            postfix.append(karakter)
            
        # 2. Jika karakter kurung buka, masukkan ke stack
        elif karakter == '(':
            stack.append(karakter)
            
        # 3. Jika karakter kurung tutup, keluarkan operator dari stack sampai ketemu kurung buka
        elif karakter == ')':
            while stack and stack[-1] != '(':
                postfix.append(stack.pop())
            stack.pop() # Buang kurung buka '(' dari stack
            
        # 4. Jika karakter adalah operator (+, -, *, /, ^)
        else:
            # Selama stack tidak kosong, elemen teratas bukan '(', dan 
            # prioritas operator saat ini <= prioritas operator di puncak stack
            while (stack and stack[-1] != '(' and 
                   prioritas.get(karakter, 0) <= prioritas.get(stack[-1], 0)):
                postfix.append(stack.pop())
            # Masukkan operator saat ini ke stack
            stack.append(karakter)
            
    # 5. Keluarkan semua sisa operator di dalam stack ke hasil
    while stack:
        postfix.append(stack.pop())
        
    return "".join(postfix)

# --- Contoh Penggunaan ---
ekspresi_infix = "A + B * (C - D)"
hasil_postfix = infix_ke_postfix(ekspresi_infix)

print(f"Ekspresi Infix   : {ekspresi_infix}")
print(f"Ekspresi Postfix : {hasil_postfix}")

Evaluasi Ekspressi Postfix¶

Seperti yang sudah dijelaskan sebelumnya, evaluasi untuk ekspressi aritmatika Postfix dilakukan setelah terdapat dua buah operand sebelum sebuah operator. Misalkan : 87+2, yang berarti (8+7)2=30, oleh karena itu berikut algoritma untuk mengevaluasi ekspressi aritmatika postfix :

  1. baca string mulai dari kiri

  2. Jika character yang dibaca adalah sebuah operand, maka push operand tersebut.

  3. Jika character yang dibaca adalah sebuah operator, maka pop dua buah operand yang terdapat pada stack, operasikan dua buah operand tersebut dengan operator, dan push hasil operasinya

  4. hasil akhir adalah angka terakhir yang terdapat di dalam stack

STACK W3¶

Stack adalah struktur data linier yang mengikuti prinsip Last-In-First-Out (LIFO).

Bayangkan seperti tumpukan pancake - Anda hanya bisa menambah atau mengurangi pancake dari bagian atas.

Tumpukan¶

Stack adalah struktur data yang dapat menampung banyak elemen, dan elemen terakhir yang ditambahkan adalah elemen pertama yang dihapus.

Seperti tumpukan pancake, pancake ditambahkan dan dikeluarkan dari atas. Jadi, ketika mengeluarkan pancake, itu selalu pancake terakhir yang Anda tambahkan. Cara mengatur elemen seperti ini disebut LIFO: Last In First Out (Terakhir Masuk, Pertama Keluar).

Operasi dasar yang dapat kita lakukan pada sebuah stack adalah:

  • Push: Menambahkan elemen baru ke dalam tumpukan.

  • Pop: Menghapus dan mengembalikan elemen teratas dari tumpukan.

  • Peek: Mengembalikan elemen teratas (terakhir) pada tumpukan.

  • isEmpty: Memeriksa apakah tumpukan kosong.

  • Ukuran: Menemukan jumlah elemen dalam tumpukan.

Stack dapat diimplementasikan dengan menggunakan array atau linked list.

Stack dapat digunakan untuk mengimplementasikan mekanisme undo, untuk kembali ke keadaan sebelumnya, untuk membuat algoritma pencarian mendalam (depth-first search) dalam graf, atau untuk penelusuran balik (backtracking).

Stack sering disebut bersamaan dengan Queue, yang merupakan struktur data serupa yang dijelaskan pada halaman berikutnya.

# Contoh Membuat tumpukan menggunakan kelas:

class Stack:
  def __init__(self):
    self.stack = []

  def push(self, element):
    self.stack.append(element)

  def pop(self):
    if self.isEmpty():
      return "Stack is empty"
    return self.stack.pop()

  def peek(self):
    if self.isEmpty():
      return "Stack is empty"
    return self.stack[-1]

  def isEmpty(self):
    return len(self.stack) == 0

  def size(self):
    return len(self.stack)

# Create a stack
myStack = Stack()

myStack.push('A')
myStack.push('B')
myStack.push('C')

print("Stack: ", myStack.stack)
print("Pop: ", myStack.pop())
print("Stack after Pop: ", myStack.stack)
print("Peek: ", myStack.peek())
print("isEmpty: ", myStack.isEmpty())
print("Size: ", myStack.size())

Alasan untuk mengimplementasikan stack menggunakan list/array:

  • Efisien Memori: Elemen array tidak menyimpan alamat elemen berikutnya seperti halnya node pada linked list.

  • Lebih mudah diimplementasikan dan dipahami: Menggunakan array untuk mengimplementasikan stack membutuhkan lebih sedikit kode daripada menggunakan linked list, dan karena alasan ini biasanya juga lebih mudah dipahami.

Alasan mengapa array tidak digunakan untuk mengimplementasikan stack:

  • Ukuran tetap: Sebuah array menempati bagian memori yang tetap. Ini berarti bahwa array tersebut dapat menggunakan lebih banyak memori daripada yang dibutuhkan, atau jika array penuh, array tersebut tidak dapat menampung elemen lagi.

Implementasi Stack menggunakan Linked List

Sebuah linked list terdiri dari node yang berisi data tertentu, dan sebuah pointer ke node berikutnya.

Salah satu keuntungan besar menggunakan linked list adalah node disimpan di mana pun ada ruang kosong di memori, node tidak perlu disimpan berdekatan satu sama lain seperti elemen yang disimpan dalam array. Keuntungan lain dari linked list adalah ketika menambahkan atau menghapus node, node lain dalam list tidak perlu digeser.

Untuk lebih memahami manfaat menggunakan array atau linked list untuk mengimplementasikan stack, Anda sebaiknya memeriksa halaman ini yang menjelaskan bagaimana array dan linked list disimpan dalam memori.

Beginilah cara mengimplementasikan stack menggunakan linked list.

#Contoh Membuat Stack menggunakan Linked List:

class Node:
  def __init__(self, value):
    self.value = value
    self.next = None

class Stack:
  def __init__(self):
    self.head = None
    self.size = 0

  def push(self, value):
    new_node = Node(value)
    if self.head:
      new_node.next = self.head
    self.head = new_node
    self.size += 1

  def pop(self):
    if self.isEmpty():
      return "Stack is empty"
    popped_node = self.head
    self.head = self.head.next
    self.size -= 1
    return popped_node.value

  def peek(self):
    if self.isEmpty():
      return "Stack is empty"
    return self.head.value

  def isEmpty(self):
    return self.size == 0

  def stackSize(self):
    return self.size

  def traverseAndPrint(self):
    currentNode = self.head
    while currentNode:
      print(currentNode.value, end=" -> ")
      currentNode = currentNode.next
    print()

myStack = Stack()
myStack.push('A')
myStack.push('B')
myStack.push('C')

print("LinkedList: ", end="")
myStack.traverseAndPrint()
print("Peek: ", myStack.peek())
print("Pop: ", myStack.pop())
print("LinkedList after Pop: ", end="")
myStack.traverseAndPrint()
print("isEmpty: ", myStack.isEmpty())
print("Size: ", myStack.stackSize())

Alasan menggunakan linked list untuk mengimplementasikan stack:

  • Ukuran dinamis: Stack dapat tumbuh dan menyusut secara dinamis, tidak seperti pada array. Alasan mengapa linked list tidak digunakan untuk mengimplementasikan stack:

  • Memori tambahan: Setiap elemen tumpukan harus berisi alamat ke elemen berikutnya (node ​​daftar tertaut berikutnya).

  • Keterbacaan: Kode ini mungkin lebih sulit dibaca dan ditulis bagi sebagian orang karena lebih panjang dan lebih kompleks.

Stack di Pyhton GFG¶

Stack adalah struktur data linier yang mengikuti prinsip Last-In/First-Out (LIFO), juga dikenal sebagai First-In/Last-Out (FILO). Ini berarti bahwa elemen terakhir yang ditambahkan adalah elemen pertama yang dihapus. Dalam sebuah stack, baik penyisipan maupun penghapusan terjadi di ujung yang sama, yang disebut puncak stack.

Python tidak memiliki tipe data stack bawaan, tetapi stack dapat diimplementasikan dengan berbagai cara menggunakan struktur data yang berbeda. Mari kita lihat beberapa implementasinya:

1. Menggunakan Daftar

  • List Python menyediakan metode bawaan yang membuatnya cocok untuk operasi stack.

  • Metode append () menambahkan elemen ke akhir daftar.

  • Metode pop() menghapus dan mengembalikan elemen terakhir dari daftar.

Operasi-operasi ini memungkinkan sebuah list untuk secara langsung mendukung perilaku seperti stack.

In [ ]:
stack = []

# append() function to push element in the stack
stack.append('a')
stack.append('b')
stack.append('c')

print('Initial stack')
print(stack)
# pop() function to pop element from stack in LIFO order
print('\nElements popped from stack:')
print(stack.pop())
print(stack.pop())
print(stack.pop())

print('\nStack after elements are popped:')
print(stack)

# uncommenting print(stack.pop()) will cause an IndexError as the stack is now empty
# Daftar dapat melambat ketika ukurannya menjadi sangat besar karena alokasi ulang memori.

2. Menggunakan collections.deque

  • Modul collections Python menyediakan deque (antrian dua ujung) untuk penyisipan dan penghapusan yang efisien.

  • Metode append () menambahkan elemen ke ujung kanan deque.

  • Metode pop() menghapus dan mengembalikan elemen dari ujung kanan.

  • Karena deque dioptimalkan untuk penambahan dan penghapusan data yang cepat, deque seringkali lebih disukai daripada list untuk implementasi stack.

class Stack:
    def __init__(self): self.items = []
    def push(self, item): self.items.append(item)
    def pop(self): return self.items.pop()
    def is_empty(self): return len(self.items) == 0

def check_with_class(text):
    s = Stack()
    for char in text:
        if char == '(': s.push(char)
        elif char == ')':
            if s.is_empty(): return False
            s.pop()
    return s.is_empty()

def remove_duplicates(s):
    stack = []
    for char in s:
        if stack and stack[-1] == char:
            continue
        stack.append(char)
    return "".join(stack)

print(remove_duplicates("aaabbcdd")) # abcd
