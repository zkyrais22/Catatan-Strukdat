Tree¶

Pohon adalah struktur data hierarkis yang terdiri dari simpul-simpul yang dihubungkan oleh sisi-sisi.

Setiap node berisi sebuah nilai dan referensi ke node anaknya.

Pohon¶

Struktur data Tree mirip dengan Linked List karena setiap node berisi data dan dapat dihubungkan ke node lain.

Sebelumnya kita telah membahas struktur data seperti Array, Linked List, Stack, dan Queue. Semua ini adalah struktur linier, yang berarti setiap elemen mengikuti elemen lain secara berurutan. Namun, Tree berbeda. Dalam sebuah Tree, satu elemen dapat memiliki banyak elemen 'berikutnya', memungkinkan struktur data untuk bercabang ke berbagai arah.

Struktur data ini disebut "pohon" karena bentuknya menyerupai struktur pohon.

Struktur data Tree dapat berguna dalam banyak kasus:

  • Data Hierarkis: Sistem file, model organisasi, dll.
  • Basis data: Digunakan untuk pengambilan data dengan cepat.
  • Tabel Perutean: Digunakan untuk merute data dalam algoritma jaringan.
  • Pengurutan/Pencarian: Digunakan untuk mengurutkan data dan mencari data.
  • Antrian Prioritas: Struktur data antrian prioritas umumnya diimplementasikan menggunakan pohon, seperti heap biner.

Jenis-jenis Pohon¶

Pohon adalah struktur data fundamental dalam ilmu komputer, yang digunakan untuk merepresentasikan hubungan hierarkis. Tutorial ini membahas beberapa jenis pohon utama.

Pohon Biner: Setiap simpul memiliki hingga dua anak, yaitu simpul anak kiri dan simpul anak kanan. Struktur ini merupakan dasar untuk jenis pohon yang lebih kompleks seperti Pohon Pencarian Biner dan Pohon AVL.

Pohon Pencarian Biner (BST): Suatu jenis Pohon Biner di mana untuk setiap simpul, simpul anak kiri memiliki nilai yang lebih rendah, dan simpul anak kanan memiliki nilai yang lebih tinggi.

Pohon AVL: Jenis Pohon Pencarian Biner yang menyeimbangkan diri sendiri sehingga untuk setiap simpul, perbedaan tinggi antara subpohon kiri dan kanan paling banyak satu. Keseimbangan ini dipertahankan melalui rotasi ketika simpul disisipkan atau dihapus.

Masing-masing struktur data ini dijelaskan secara detail pada halaman-halaman berikutnya, termasuk animasi dan cara mengimplementasikannya.

Pohon vs. Array dan Daftar Berantai¶

Keunggulan Struktur Pohon dibandingkan Array dan Linked List:

  • Array cepat ketika Anda ingin mengakses elemen secara langsung, seperti elemen nomor 700 dalam array yang berisi 1000 elemen misalnya. Tetapi memasukkan dan menghapus elemen memerlukan pergeseran elemen lain di memori untuk memberi tempat bagi elemen baru, atau untuk menggantikan elemen yang dihapus, dan itu memakan waktu.
  • Linked list cepat dalam menyisipkan atau menghapus node, tidak memerlukan pergeseran memori, tetapi untuk mengakses elemen di dalam list, list harus ditelusuri, dan itu membutuhkan waktu.
  • Pohon , seperti Pohon Biner, Pohon Pencarian Biner, dan Pohon AVL, sangat bagus dibandingkan dengan Array dan Daftar Bertautan karena pohon CEPAT dalam mengakses sebuah node, DAN cepat dalam menghapus atau menyisipkan sebuah node, tanpa perlu pergeseran memori.

Beberapa istilah pada Tree yang harus diketahui adalah sebagai berikut :

  • Node, merupakan bagian dari tree yang menyimpan informasi atau juga disebut sebagai key
  • Edge, yang menghubungkan antara node yang satu dengan yang lain. Hanya ada satu edge yang masuk ke dalam node, dan satu atau lebih edge yang keluar dari suatu node
  • Root, node yang terletak di paling atas, tidak ada edge yang masuk ke dalam node root ini, tapi root mempunyai satu atau lebih dari satu edge yang keluar.
  • path, merupakan urutan node dari root sampai tujuan, misalkan computer --> D: --> Mata Kuliah --> Struktur Data
  • children, merupakan node-node yang memiliki edge yang masuk dari node yang sama. Misalkan program files, users, dan windows merupakan children, krn memiliki edge masuk yang sama-sama berasal dari node C:
  • Parent, node yang memiliki edge yang keluar dari node tersebut, misalkan node C: adalah parent dari program files, users, dan windows
  • Siblings, adalah node-node yang berasal dari parent yang sama
  • subtree, kumpulan dari node dan edge yang terdiri dari parent dan semua node setelah parent
  • leaf node, node yang tidak memiliki edge yang keluar, node yang tidak memiliki children level, tingkatan dari node. Root merupakan level ke-0, semakin kebawah maka level dari
  • node tersebut semakin besar, misalkan struktur data berada di level ke-3
  • height, merupakan tinggi tree, nilai dari height ini adalah level maksimum dari tree
